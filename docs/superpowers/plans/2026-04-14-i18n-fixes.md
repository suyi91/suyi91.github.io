## Task A: Fix HeaderLink Active State

**Files:**
- Modify: `src/components/HeaderLink.astro`

### Root Cause

Line 7: `const isActive = hrefMath[1] === pathnameMatch[1]`

For `/zh/blog/1`, `pathnameMatch[1]` is `'zh'`. Every zh-prefixed href also has `'zh'` at index 1 → **all nav links are simultaneously active**.

### Fix

Replace the buggy comparison with a `getSection()` helper that strips the lang prefix before comparing.

- [ ] **Step 1: Edit `src/components/HeaderLink.astro`**

Replace the entire frontmatter block (lines 1–8):

```astro
---
const {href, icon: iconClass, children, target, ...props} = Astro.props;

const {pathname} = Astro.url;

function getSection(path: string): string {
  const parts = path.split('/').filter(Boolean)
  const first = parts[0]
  return (first === 'zh' || first === 'en') ? (parts[1] ?? '') : first
}

const isActive = getSection(href) === getSection(pathname)
---
```

## Task C: Fix formatDate + PostTitle + PostViewTitle + PostView Lang-Awareness

**Files:**
- Modify: `src/utils/formatDate.ts`
- Modify: `src/components/PostTitle.astro`
- Modify: `src/components/PostViewTitle.astro`
- Modify: `src/components/PostView.astro`
- Modify: `src/pages/blog/[...slug].astro`
- Modify: `src/pages/zh/blog/[...slug].astro`

### Root Cause

`formatDate.ts` uses a module-level `t` bound to `config.lang` at import time — it ignores the actual page language. `zhCn.ts` also had no `'post.dateFormat'` key (fixed in Task B). `PostTitle` and `PostViewTitle` have no `lang` prop and pass nothing to `formatDate`. The category/tag links in `PostTitle`/`PostViewTitle` always use `/category/` prefix regardless of lang.

### Fix

Add `lang: Lang` param to `formatDate`. Add `lang` prop to `PostTitle` and `PostViewTitle`. Fix category/tag link prefixes. Pass `lang` explicitly from callers.

- [ ] **Step 1: Rewrite `src/utils/formatDate.ts`**

Replace the entire file:

```ts
import dayjs from "dayjs";
import advancedFormat from 'dayjs/plugin/advancedFormat'
import utc from "dayjs/plugin/utc";
import { useTranslations, type Lang } from '../i18n/utils';
import { config } from '../consts';

dayjs.extend(advancedFormat)
dayjs.extend(utc);

export function formatDate(date: Date | string | undefined, lang: Lang = 'en', dateType = 'post.dateFormat'): string {
  if (!date) return ''
  const t = useTranslations(lang)
  const dateFormat = t(dateType) || 'YYYY-MM-DD'
  return dayjs(date).utc().format(dateFormat)
}
```

Note: `dayjs.locale(config.lang)` is removed — it relied on a global locale that didn't affect `.format()` output anyway. The `config` import is no longer needed.
